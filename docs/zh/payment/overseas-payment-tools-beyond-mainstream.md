# Stripe 和 PayPal 之外：那些你可能没听过的海外收款工具

Stripe 好用但要公司，PayPal 门槛低但体验差。这两个之外，市场上还有一堆收款工具，有些是专门给独立开发者设计的，有些解决特定场景的问题。

这篇把它们分类梳理一遍。重点讲每个工具的定位、费率、以及对中国开发者是否友好。

## MoR 平台：帮你搞定税务的「全托管」方案

MoR 是 Merchant of Record 的缩写，意思是这个平台以它自己的名义向用户收款，帮你处理全球销售税（VAT / GST / Sales Tax），然后把扣除费用和税款后的净额打给你。

你不需要自己注册公司、不需要搞 VAT 号、不需要在欧盟 27 个国家分别报税。平台全包了。代价是费率比 Stripe 高。

### LemonSqueezy

**定位：** 专为数字产品和 SaaS 设计的 MoR 平台，2022 年被 Stripe 收购后仍独立运营。

| 项目 | 详情 |
|------|------|
| 费率 | 5% + $0.50/笔 |
| 支持产品类型 | 数字下载、SaaS 订阅、License Key、付费邮件 |
| 税务处理 | 全球 VAT/GST/Sales Tax 自动计算和代缴 |
| 提现方式 | PayPal、银行转账（支持多国） |
| 对中国开发者 | 支持中国个人注册，用护照验证身份 |

**优点：** 不需要公司实体就能注册；后台 UI 简洁；支持 License Key 分发；被 Stripe 收购后稳定性有保障。

**缺点：** 费率高（5% + $0.50 vs Stripe 的 2.9% + $0.30）；提现到国内需要通过 PayPal 或 Payoneer 中转；大额交易时费率劣势更明显。

**适合谁：** 没有海外公司、不想折腾税务、卖数字产品或低价 SaaS（月费 $10-30）的独立开发者。如果你的月收入超过 $3000，费率差额就值得你花 $500 注册个 LLC 用 Stripe 了。

### Paddle

**定位：** 面向 SaaS 和软件公司的 MoR 平台，比 LemonSqueezy 更偏企业级。

| 项目 | 详情 |
|------|------|
| 费率 | 5% + $0.50/笔（标准），大客户可谈判 |
| 支持产品类型 | SaaS 订阅、桌面软件 License、一次性购买 |
| 税务处理 | 全球税务合规（200+ 国家） |
| 提现方式 | 银行转账（USD/EUR/GBP） |
| 对中国开发者 | 需要公司实体（不支持个人注册） |

**和 LemonSqueezy 的区别：** Paddle 不支持个人注册，需要公司身份。但它的 checkout 体验更好、支持更多支付方式（Apple Pay、Google Pay、Wire Transfer）、有专属客户经理（达到一定交易量后）。适合已有公司、交易量较大的 SaaS 产品。

### FastSpring

**定位：** 老牌 MoR，主要服务软件和游戏行业。

| 项目 | 详情 |
|------|------|
| 费率 | 5.9% + $0.95/笔（据公开资料，具体需联系销售） |
| 支持产品类型 | 软件 License、订阅、游戏内购 |
| 对中国开发者 | 需要公司实体 |

FastSpring 费率最高，但在桌面软件和游戏领域有很深的积累。如果你做的是一次性买断的桌面工具（比如 macOS 效率软件），FastSpring 的 License 管理功能比较成熟。对大多数做 Web SaaS 的独立开发者来说，不是第一选择。

## 轻量收款工具：门槛最低的起步方案

### Gumroad

**定位：** 卖数字产品的极简平台。

| 项目 | 详情 |
|------|------|
| 费率 | 10%（含支付处理费） |
| 支持产品类型 | 电子书、模板、课程、软件、会员 |
| 提现方式 | PayPal、Stripe（连接你自己的 Stripe 账号可降低费率） |
| 对中国开发者 | 支持个人注册 |

**现状：** Gumroad 在 2023 年大幅调整了费率（从阶梯制改为统一 10%），很多创作者流失到了 LemonSqueezy 或 Polar。10% 的费率确实偏高，但如果你只是验证一个数字产品能不能卖（比如一套 Notion 模板、一个 Figma UI Kit），Gumroad 的注册和上架流程可能是最快的——10 分钟就能开始卖。

### Ko-fi / Buy Me a Coffee

**定位：** 打赏和小额付费。

| 平台 | 费率 | 特色 |
|------|------|------|
| Ko-fi | 0%（免费版）/ 5%（Gold 会员功能） | 不抽成（免费版），支持打赏、会员、Shop |
| Buy Me a Coffee | 5% | UI 更好看，支持打赏、会员、Extras |

这两个的定位很接近：让你的粉丝/用户通过「请你喝杯咖啡」的方式小额打赏，或者按月订阅支持你。

**对中国开发者：** 都支持个人注册，通过 Stripe Connect 或 PayPal 收款。如果你有 Stripe 账号，Ko-fi 免费版完全不抽成（0%），只有支付网关本身的费率（Stripe 2.9% + $0.30）。

**适合谁：** 开源项目维护者、博主、工具作者。不适合做主力收入来源，但作为补充收入不错。

### Polar.sh

**定位：** 专门为开源开发者设计的商业化平台。

| 项目 | 详情 |
|------|------|
| 费率 | 5%（Polar 抽成）+ 支付处理费 |
| 功能 | GitHub Sponsors 替代方案、付费订阅、License Key、付费功能解锁 |
| 对中国开发者 | 支持个人 GitHub 账号注册 |

Polar 2024 年才上线，但在开源社区增长很快。它解决的是一个很具体的问题：GitHub Sponsors 对非美国开发者提现不方便，而且功能太简陋（只能收打赏，没有 License 管理）。Polar 让你可以把开源项目的某些功能、文档或者支持服务做成付费产品，同时保持核心代码开源。

### Creem

**定位：** 2024 年新出的 MoR 平台，主打对独立开发者友好。

| 项目 | 详情 |
|------|------|
| 费率 | 4% + $0.40/笔 |
| 特色 | 比 LemonSqueezy 便宜、API 简洁、专注数字产品 |
| 对中国开发者 | 支持个人注册（据其官网，截至 2026 年 6 月） |

Creem 是 LemonSqueezy 的竞品，费率更低（4% vs 5%），功能覆盖数字产品和 SaaS 订阅。2025 年开始在独立开发者社区有一定声量。缺点是平台较新，稳定性和长期存续性还有待验证。

## 加密货币收款

如果你的用户群体里有 crypto 持有者（Web3 开发者、交易员、矿工），提供加密货币支付选项可以降低他们的付款摩擦。

**常见方案：**

| 方案 | 说明 | 费率 |
|------|------|------|
| 直接收 USDT/USDC | 给用户一个钱包地址，手动确认 | 0%（只有链上 gas 费） |
| Coinbase Commerce | 接入 API，支持 BTC/ETH/USDC 等 | 1% |
| NOWPayments | 支持 100+ 币种，自动换算法币 | 0.5%-1% |
| BTCPay Server | 开源自托管方案，无平台费 | 0%（自己承担服务器成本） |

**对中国开发者的注意事项：**
- 收到的加密货币如果想换成人民币，需要通过交易所 OTC 或者 P2P，存在合规风险
- 如果只是持有 USDC/USDT 然后通过 Wise 换汇，链路更清晰（USDC → Coinbase → USD → Wise → CNY）
- 加密收款适合作为补充选项，不适合作为唯一收款方式——大部分用户还是习惯信用卡

## 怎么选：一张决策表

| 你的情况 | 推荐方案 |
|---------|---------|
| 有美国 LLC，做 SaaS | Stripe（主力）+ PayPal（备选） |
| 有公司，想省税务麻烦 | Paddle 或 LemonSqueezy |
| 没有公司，卖数字产品 | LemonSqueezy 或 Creem |
| 没有公司，只想试水 | Gumroad 或 Ko-fi |
| 开源项目想商业化 | Polar.sh 或 GitHub Sponsors |
| 用户群有 crypto 持有者 | Coinbase Commerce（补充） |
| 月收入 < $500 | 用免费方案（Ko-fi）或低门槛方案（Gumroad） |
| 月收入 > $3000 | 值得注册 LLC + 用 Stripe，费率差额能回本 |

## 一个实际的演进路径

大多数独立开发者的收款方案会经历这样一个过程：

**阶段一（验证期）：** 产品刚上线，不确定能不能卖出去。用 LemonSqueezy 或 Gumroad，5-10 分钟接入，先跑通第一笔订单。这个阶段费率高不高无所谓，重要的是验证付费意愿。

**阶段二（增长期）：** 月收入稳定在 $1000 以上，开始在意费率。注册美国 LLC，切换到 Stripe。每月省下的费率差额（$1000 × 2% ≈ $20/月）虽然不多，但 Stripe 的功能（订阅管理、发票、webhook）对产品迭代更友好。

**阶段三（规模期）：** 月收入超过 $10000，开始面对全球税务合规问题。要么自己搞定各国 VAT 注册（复杂），要么用 Paddle/LemonSqueezy 的 MoR 功能把税务外包（省心但贵）。这时候需要算一笔账：自己请会计处理税务的成本 vs MoR 平台的额外费率。

不需要一步到位。先用门槛最低的方案把钱收进来，再根据收入规模逐步升级。
