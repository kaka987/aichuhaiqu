# 卖信息产品：电子书、课程、模板的数字商品生意

不是每个人都要做 SaaS。有一种变现方式，不用维护服务器、不用做客服、不用担心用户用太多把你成本拖垮——卖信息产品。

信息产品就是把你的知识、经验、做好的东西，打包成可以无限复制的数字商品来卖：电子书、视频课程、Notion 模板、代码模板、设计素材、提示词库。它最大的特点是**边际成本接近零**——做出来要花时间，但之后卖一份和卖一千份，成本几乎一样。

很多独立开发者的第一桶金，甚至长期主要收入，来自信息产品而不是软件。Daniel Vassallo 靠一本 AWS 电子书和一门 Twitter 课程做到百万美金（见 [他的故事](maker-stories/daniel-vassallo-small-bets.md)）；Marc Lou 的 ShipFast 本质也是个代码模板（见 [ShipFast 拆解](case-studies/shipfast-selling-shovels.md)）。

## 为什么它适合中国开发者起步

- **成本极低**：不需要服务器、不需要持续的 API 开销。做一次，长期卖
- **没有交付负担**：用户付款后自动下载，没有客服、没有运维
- **把已有的东西变现**：你工作中、做项目时积累的经验和工具，本来就在你脑子里/硬盘里，打包出来就能卖
- **快**：一个 SaaS 要做几个月，一本电子书、一套模板可能一两周就能上线

对预算有限、想低成本验证「我能不能从陌生人手里赚到钱」的人，这是门槛最低的一条路。

## 卖什么：从你已有的东西里找

信息产品的内容，最好来自你**已经会、已经做过**的东西。常见的几类：

- **电子书 / 指南**：把某个领域的实操经验系统化。比如「中国开发者出海收款完全指南」「Stable Diffusion 商用出图手册」
- **代码模板 / 启动套件**：把重复造的轮子打包。SaaS 模板、落地页模板、特定功能的代码
- **Notion / Excel 模板**：把你的工作流、管理系统模板化。这类在海外卖得很好
- **课程 / 教程**：录成视频，教一个具体技能
- **设计素材 / 资源包**：图标、UI kit、字体、提示词库
- **AI 提示词 / 工作流**：把调教好的 prompt、自动化工作流打包

判断标准：**有没有一群人，正在为「不会做这个」而头疼，且愿意花钱跳过学习过程。** 你卖的是他们省下的时间和踩坑成本。

## 用什么平台卖和收款

对中国开发者，平台选择的关键是**能不能顺利收到钱**。几个主流选择（截至 2026 年，以各平台政策为准）：

- **Gumroad**：独立创作者卖数字商品的老牌平台，上手简单，自带支付。抽成比例以官网为准
- **Lemon Squeezy / Paddle**：作为 Merchant of Record（MoR），帮你处理全球税务（VAT、销售税），省去自己报税的麻烦，特别适合不想碰复杂税务的个人
- **自建 + Stripe**：用 Stripe 收款，自己挂个落地页卖。控制力强，但税务要自己处理

MoR 平台（Lemon Squeezy、Paddle、Gumroad 部分模式）对个人最友好，因为它们替你当「法律上的卖家」，处理增值税和合规问题。具体收款通道的选择见 [Stripe/PayPal 之外的收款工具](payment/overseas-payment-tools-beyond-mainstream.md)。

## 怎么定价

信息产品定价比 SaaS 灵活，因为没有持续成本压力。几个常见区间（仅供参考）：

- **小型电子书 / 模板**：$9-49，冲动消费区间，决策快
- **深度指南 / 启动套件**：$49-199，有明确实用价值的
- **课程 / 高价值套件**：$99-499+，解决具体高价值问题（比如 ShipFast 的 $199-299）

定价的核心不是「内容多少」，而是**「帮用户省了多少 / 赚了多少」**。一套帮开发者省下两周开发时间的模板，卖 $199 也很值，因为两周时间远不止这个价。别按字数和页数定价，按价值定价。

也可以用「锚点」策略：出两三个档位（基础版 / 完整版 / 含咨询版），大多数人会选中间那档，这本身能拉高客单价。

## 真正的门槛：你得先有人听你说话

信息产品听起来美好——做一次卖无数份。但有个绕不开的前提：**没有受众，再好的信息产品也卖不动。**

为什么 Daniel Vassallo、Marc Lou 能靠信息产品赚钱？因为他们在卖之前，已经在 Twitter 上积累了成千上万信任他们的关注者。当他们推出产品时，有现成的人愿意买、愿意传播。

这是信息产品和 SaaS 最大的区别：
- SaaS 可以靠 SEO、靠功能本身慢慢获客
- 信息产品高度依赖**信任**——人们买的是「你这个人的经验」，得先相信你

所以如果你现在零受众，更现实的路径是：
1. 先在某个平台（X / 小红书 / 知乎 / Indie Hackers）持续分享你的专业内容，积累信任
2. 同时把这些内容沉淀成体系
3. 受众积累到一定程度，再把体系打包成产品卖给他们

**内容是免费版，信息产品是付费版。** 你平时免费分享的东西建立信任，把最系统、最有价值的部分打包收费。

## 对中国开发者的现实建议

**1. 从你已经会的东西开始，别为了卖去现学。** 你工作和项目里的真实经验，就是最好的素材。

**2. 用 MoR 平台收款省心。** Lemon Squeezy、Paddle 这类帮你处理全球税务，个人卖家不用头疼增值税。

**3. 按价值定价，不按字数。** 算清楚你帮用户省了多少时间/钱，据此定价。

**4. 先攒受众，再卖产品。** 这是信息产品成败的关键。零受众别指望卖得动，先用内容建立信任（参考 [有钱投流没钱做号](growth/paid-vs-organic-traffic-strategy.md)）。

**5. 可以和其他变现叠加。** 信息产品 + 联盟（在内容里推荐工具）+ 后续的 SaaS，是很自然的组合。Daniel Vassallo 的「一堆小赌注」就是这个思路。
