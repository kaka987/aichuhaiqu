# 不卖产品也能赚钱：打赏、赞助与影响力变现

有一类东西很难直接「卖」——一个免费的开源工具、一个写得好的博客、一个活跃的社区。它们有价值、有人用、有影响力，但没有一个明确的「付费按钮」。

这类资产的变现方式，是把**影响力和信任本身**变成钱：靠赞助、打赏、付费会员。它的逻辑和卖产品完全不同——你不是在卖一个东西，你是在让那些从你这里获得价值的人，自愿地支持你继续做下去。

知识库里拆解过的 CC Switch 就是典型：一个 MIT 开源、完全免费的工具，靠 10 万多 Star 的影响力撬动了 20 多家赞助商（见 [CC Switch 变现拆解](case-studies/cc-switch-monetization-playbook.md)）。

## 这条路的核心前提：先有影响力

把话说在前面，免得你抱有不切实际的期待：**赞助和打赏，是「先有名、再赚钱」的变现方式。**

没有人会赞助一个没人用的项目、打赏一个没人看的博客。这条路的全部前提，是你已经积累了某种影响力：
- 一个被很多人使用的开源项目
- 一批信任你的读者/关注者
- 一个活跃的社区

如果你还没有这些，这篇可以先了解，但当务之急是先把影响力做起来（通过持续做好东西、持续分享）。影响力是这一切的本金。

## 主流的几种方式

### 1. 开源项目赞助

如果你维护一个有人用的开源项目，这是最自然的变现：

- **GitHub Sponsors**：GitHub 官方的赞助功能，个人和企业都能按月赞助你。门槛低、信任度高，是开源作者的首选
- **Open Collective**：更适合有多个贡献者的项目，资金流向公开透明
- **README 赞助商位**：在项目主页（README）放赞助商的 logo 和链接。Star 多、流量大的项目，这个位置很值钱——CC Switch 走的就是这条
- **Buy Me a Coffee / Ko-fi**：轻量的打赏工具，用户一次性「请你喝杯咖啡」

开源赞助的现实：**大部分赞助来自企业，不是个人。** 个人打赏金额小、随机；真正的钱通常来自那些「依赖你的项目、希望它持续维护」的公司。所以一个被商业项目广泛使用的工具，赞助潜力远大于一个个人玩具项目。

### 2. 内容创作者的赞助

如果你做内容（博客、newsletter、YouTube、播客），变现方式包括：

- **内容赞助 / 植入**：品牌付钱在你的内容里露出（newsletter 里的一段赞助、视频里的口播）。按受众规模和精准度定价
- **Newsletter 赞助位**：邮件列表如果足够垂直、打开率高，赞助位很好卖。出海/技术类的 newsletter 尤其受 B2B 工具青睐
- **打赏 / 会员**：让喜欢你内容的人按月支持，作为回报给一些额外内容或社区权限

### 3. 付费社区 / 会员制

当你的影响力聚集起一群人，可以把「人聚在一起」本身做成产品：

- **付费社区**：按月/按年收费，提供一个高质量的同好圈子 + 你的持续参与。Daniel Vassallo 的 Small Bets 社区就是典型，累计做到数十万美金后卖给了 Gumroad（见 [他的故事](maker-stories/daniel-vassallo-small-bets.md)）
- **会员订阅**：Patreon 式的分级会员，不同档位给不同权益

付费社区的关键是**持续的价值和氛围**，而不是一锤子买卖——它更像订阅，要持续运营，否则会员会流失。

## 收款要注意的点

这些钱大多以美金结算，对中国开发者，收款通道要提前安排：

- GitHub Sponsors、Ko-fi、Patreon 等通常通过 Stripe / PayPal / 银行电汇结算
- 确认你有合规的收款和回款路径（见 [合法回款](payment/bringing-overseas-earnings-back-to-china.md)）
- 企业赞助可能需要开发票/签协议，这时有个海外主体会方便很多（见 [为什么注册海外公司](overseas-company/why-overseas-company-for-ai.md)）

## 这条路的优缺点要看清

**优点：**
- 不用做付费产品，免费的东西也能变现
- 和你「做好东西、做好内容」的本职高度一致，不用额外做销售
- 企业赞助一旦建立，可能相当稳定

**缺点：**
- **收入不稳定、不可预测**：赞助靠的是人情和善意，没有合同保障，说停就停
- **强依赖影响力**：天花板由你的影响力决定，影响力涨它才涨
- **金额通常有限**：除非影响力非常大，否则纯赞助很难支撑全职收入

## 现实的定位：组合里的一条，不是全部

对绝大多数人，赞助和打赏**不适合作为唯一收入来源**——它太不稳定。更现实的定位是把它作为多元收入里的一条：

- 开源项目：赞助 + 付费 Pro 版 + 付费技术支持
- 内容创作者：赞助 + 联盟营销 + 卖信息产品
- 社区：会员费 + 信息产品 + 线下活动

CC Switch 的价值也在这里——它证明了「一个不能直接卖钱的免费产品，可以靠影响力走出一条变现路」。但这条路的本金，永远是你先得有影响力。

## 对中国开发者的建议

**1. 先做出有人用、有人信的东西。** 影响力是这条路的全部前提，没有捷径。

**2. 开源项目优先盯企业赞助。** 个人打赏是零花钱，企业赞助才是正经收入。让你的项目被商业场景广泛使用，赞助价值才高。

**3. 把它当组合的一部分。** 别指望纯赞助养活自己，和付费版、信息产品、联盟叠加才稳。

**4. 收款和主体提前准备。** 企业赞助常需要协议和发票，有海外主体会顺畅很多。
